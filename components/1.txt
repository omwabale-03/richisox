Build a complete production-ready Next.js 14 e-commerce website for a sock brand called RichySox (domain: RichySox.com). This is a Direct-to-Consumer (D2C) business selling premium socks for men, women, and kids across India.

## Tech Stack
- Next.js 14 with App Router + TypeScript
- Tailwind CSS for styling
- MongoDB + Mongoose for database
- Zustand for state management (cart + auth), persisted to localStorage
- JWT for authentication
- Razorpay for payments (Indian payment gateway — UPI, cards, wallets, net banking)
- MSG91 for OTP SMS (or Firebase Auth as fallback)
- Shiprocket for shipping and order tracking
- Vercel for frontend hosting
- MongoDB Atlas for database hosting
- react-hot-toast for notifications
- react-icons for icons
- Google Fonts: Playfair Display (headings) + DM Sans (body)

## Brand Identity
- Brand name: RichySox
- Logo style: "Richy" in black + "Sox" in gold (#c9a84c)
- Colors: Black (#0a0a0a), Gold (#c9a84c), Cream (#f4f0e8), White (#fafaf7)
- Vibe: Luxury + fun + colorful + bold — all at once
- Font pairing: Playfair Display (serif, for headings) + DM Sans (clean, for body)
- Design language: Rounded cards (border-radius 16–24px), minimal borders, generous whitespace, gold accents

## Authentication System
- Mobile OTP login only — no passwords
- Flow: Enter mobile → receive SMS OTP → verify → logged in or new account created
- JWT token stored in httpOnly cookie + Zustand store
- MSG91 integration for SMS (with fallback: print OTP to console in dev mode when API key is empty)
- Auto-save address for returning customers

## Business Logic Details

### Pricing
- Free shipping on orders ≥ ₹499, else ₹49 shipping fee
- Coupon types: percentage (with optional maxDiscount cap) or flat amount
- Order ID format: RS-YYYYMMDD-0001 (auto-incremented, generated in pre-save hook)

### Payment Flow
1. POST /api/orders → creates order with status "pending", reduces stock
2. POST /api/payment/create-order → creates Razorpay order, saves razorpayOrderId
3. Razorpay popup opens in browser
4. On success: POST /api/payment/verify → verifies HMAC signature → updates order to paid+confirmed → calls Shiprocket API to create shipment

### OTP Flow
- generateOTP(): 6-digit random number
- storeOTP(): saves to in-memory Map with 5-minute expiry
- verifyOTP(): checks Map, deletes after successful verify
- sendOTP(): calls MSG91 API if AUTH_KEY present, else console.log for dev
- On verify: find user by mobile or create new one, return JWT

### Admin Auth
- All /api/admin/* routes check getUserFromRequest → must have role === "admin"
- Admin sidebar checks isAdmin() from Zustand store, redirects to / if not admin

## Additional Requirements
- All API routes return { success: boolean, data?, message?, error? }
- Soft delete products (set isActive: false, never hard delete)
- Cart persisted in localStorage via Zustand persist middleware
- Auth token stored in both httpOnly cookie (for SSR) and Zustand store (for client)
- MongoDB connection cached on global object to survive Next.js hot reloads
- Product full-text search index on name + description + tags fields
- Order stock reduction happens at order creation, not at payment
- Shiprocket order creation is non-fatal (wrapped in try/catch, order still succeeds if Shiprocket fails)
- tailwind.config.ts must include fontFamily for playfair and sans, plus marquee keyframe animation
- globals.css must include skeleton shimmer animation and custom scrollbar styles
- Deliver all files as a valid ZIP using Python zipfile module (not shell zip command)
